# Self-Demo Guide

> **SynTree KV** 프로젝트 웹사이트 시연 가이드  
> 이 문서를 따라 단계별로 웹사이트를 둘러보며 프로젝트의 핵심 내용을 파악할 수 있습니다.

---

## 사전 준비

### 1. 웹사이트 접속

브라우저에서 아래 주소로 접속합니다.

```
https://def-omega.vercel.app/
```

권장 환경:
- **데스크톱**: Chrome, Safari, Firefox 최신 버전 (창 너비 1100px 이상 권장)
- **모바일**: iOS Safari, Android Chrome (햄버거 메뉴 자동 활성화)

### 2. 화면 확인

접속 직후 다음이 보여야 정상입니다.

- 화면 상단 좌측: `● SynTree KV` 로고 
- 화면 상단 우측: `GitHub →` 버튼 (모바일에서는 햄버거 메뉴 `≡`)
- 메인 히어로 영역: 큰 제목 `SynTree KV` 표시
- 핵심 수치 카드 4개: `83.3%` / `42.4%` / `48개` / `GO ✓`

## 네비게이션

- **데스크톱**: 상단 메뉴 클릭 → 해당 섹션으로 이동
- **모바일**: 우측 상단 `≡` 버튼 → 메뉴 펼치기 → 항목 클릭 → 자동으로 닫힘

스크롤하면 현재 위치 섹션이 메뉴에서 자동으로 강조됩니다.

## 시연 순서

본 웹사이트는 **7개의 섹션**으로 구성되어 있으며 아래 순서대로 확인하면 프로젝트의 배경부터 실험 결과까지 핵심 내용을 이해할 수 있습니다.

### 01. 문제 정의
코딩 에이전트 환경에서 발생하는 KV Cache 재계산 문제를 확인합니다.

**확인 포인트**
- 코드 일부만 수정되어도 전체 Prefix를 다시 계산해야 하는 기존 방식의 한계
- 반복적인 코드 수정 과정에서 발생하는 불필요한 추론 비용
- 코딩 에이전트 워크플로우에서 KV 재사용이 필요한 이유

### 02. 방법론
세 가지 핵심 아이디어를 카드로 확인합니다.
- ① AST 서브트리 단위 KV 재사용
- ② α-rename 정규화 + BLAKE3 지문
- ③ RoPE Re-positioning

**확인 포인트**
- 코드 일부만 수정되어도 변경되지 않은 영역의 KV를 재사용하는 방법
- 의미적으로 동일한 코드를 안정적으로 식별하는 방법
- 재사용된 KV를 새로운 위치에서도 활용할 수 있도록 하는 기법

### 03. 실험 결과
MVE 실험 수치를 확인합니다.
- Hit Rate 4가지 조건 모두 **GO** 판정
- Vanilla vs Prefix Caching TTFT 비교 그래프 2장

**확인 포인트**
- Hit Rate 목표 기준 충족 여부
- Prefix Caching 대비 KV 재사용 성능 향상 여부
- TTFT(Time To First Token) 감소 효과
- 제안 기법의 실효성 검증 결과

### 04. 실험 환경
서버 구성과 기술 스택을 확인합니다.
- ewha-acpl3 서버 · RTX 5090 × 2
- OpenHands 플로우 다이어그램

**확인 포인트**
- 실험에 사용된 하드웨어 환경
- OpenHands 기반 코딩 에이전트 워크플로우
- 실험 재현을 위한 시스템 구성

### 05. 시스템 구조
전체 파이프라인 6단계를 확인합니다.

**확인 포인트**
- 코드 입력부터 AST 생성까지의 흐름
- 정규화 및 Fingerprint 생성 과정
- KV Cache 조회 및 재사용 과정
- 최종 추론 단계까지의 전체 파이프라인

### 06. 향후 계획
완료된 작업과 남은 작업을 로드맵 카드로 확인합니다.

**확인 포인트**
- 현재 완료된 개발 항목
- 남은 구현 과제
- 성능 개선 및 확장 계획
- 실제 코딩 에이전트 적용 방향

### 07. 팀 소개
팀원 및 지도교수 정보를 확인합니다.

**확인 포인트**
- 프로젝트 참여 인원
- 팀 구성
- 지도교수 정보

## 기술 정보

- **단일 HTML 파일** — 외부 빌드 도구나 프레임워크 없이 `index.html` 하나로 구성
- **외부 의존성** — Google Fonts (DM Serif Display, DM Mono, Noto Sans KR) 만 사용
- **이미지** — base64 인라인 임베드로 외부 파일 불필요
- **반응형** — 768px 이하에서 모바일 레이아웃 자동 전환
- **호스팅** — Vercel (자동 CDN, HTTPS 적용)
